<h2 id="about-the-project">About Quick Photo Editor</h2>

<div align="center">
  <img alt="display" src="https://raw.githubusercontent.com/ZiaCodes/img_server/main/others/Screenshot%202021-09-30%20023614.jpg" width="650" height="350">

<br>

Put together from the efforts of open source contributors from all over the world, Quick Photo Editor is a quick online editing tool to perform on-the-go editing like aspect adjustion, contrast and exposure tweaking etc.

<h2 id="getting-started">Getting Started</h2>

1. Fork the repository.
![image](https://user-images.githubusercontent.com/67458613/136273792-ba606639-16de-48a0-9a00-77670382f156.png)

2. Clone the repository into your local environment.

- From git CLI
  ```sh
  git clone https://github.com/gdscjgec/Image-Editor.git
  ```
  
_If you're not comfortable with command line, [here is a comprehensive tutorial for beginners by freeCodeCamp](https://www.youtube.com/watch?v=RGOj5yH7evk)._<br>
_If you don't have git on your machine, [install it](https://help.github.com/articles/set-up-git/)._

<h2 id="contributing">Contributing</h2>

Contributions are what make the open source community such an amazing place to learn, inspire, and create. Any contributions you make are **greatly appreciated**. Please make sure to follow these steps to make a contribution:

1. Raise an issue [here](https://github.com/gdscjgec/Image-Editor/issues) to address the change being made you.<br>

2. Fork the repository using the `fork` button at the top-right corner of the screen.<br>

3. Clone the repository into your local environment to make the changes in.<br>

4. Create and switch to your feature branch to make the changes.<br>
```sh
git checkout -b feature/AmazingFeature
```

4. After writing the required code, add and commit your changes to your feature branch.<br>
```sh
git add .
git commit -m "Add this AmazingFeature"
```

5. Push the commited changes to the feature branch in your remote repository.<br>
```sh
git push origin feature/AmazingFeature
```

7. Open a pull request by comparing your feature branch with the main branch of the project (remember to link your issue with the pull request).<br>

8. Wait for a maintainer to review and merge your commit into the project or ask for changes if required.<br>

**NOTE:** Kindly read our [contributing rules](https://github.com/gdscjgec/Image-Editor/blob/main/CONTRIBUTING.md) before submitting a request!
